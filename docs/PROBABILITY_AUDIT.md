# Probability, RNG & Rules-Fidelity Audit — Blackjack Trainer

**Scope:** `src/engine.js`, `src/rules.js`, `src/evdata.js`, `src/App.jsx` (game + Coach Me money paths), test suite.
**Reference ruleset:** Atlantic City 8-deck — dealer stands on all 17s (S17), blackjack pays 3:2, dealer peeks
under 10/A, double on any two cards, double after split (DAS), split to 4 hands, split aces get one card,
late surrender.
**Date:** 2026-07-07. Audited on branch `claude/casino-gateway-probability-check-cpfzy1`.

## Methodology

1. **Code review** of the shuffle/deal path, settlement paths (`finalizeOpening`, `resolveRound`, the
   surrender branches in `playerAct`/`coachAct`, insurance resolution), and the strategy/EV tables.
2. **Theory cross-check** of the basic-strategy matrix, surrender set, Illustrious-18/Fab-4 indices and
   `evdata.js` EVs against Wizard of Odds and Schlesinger (*Blackjack Attack*) values for 8-deck S17 DAS LS.
3. **Monte Carlo** (throwaway script, run via `vite-node`, not committed): 10,000,000 full rounds played by
   the engine's own `getPlay` basic strategy (with late surrender, no insurance) against the engine's own
   shoe/cut-card cycle, plus RNG chi-square tests and per-round card-conservation asserts.
4. **Test suite:** `npx vitest run` before and after changes.

## Findings

| # | Area | Severity | Finding | Status |
|---|------|----------|---------|--------|
| 1 | RNG (live path) | **High** | `rnd()` used `Math.floor(Math.random() * n)` for the live shuffle and every deal. `Math.random()` is not cryptographically sound and is seed-observable in principle. (Fisher–Yates loop itself was correct; `floor(random()*n)` has no material modulo bias, so this was a *quality-of-source* issue, not a distribution bug.) | **Fixed** — `rnd()` now draws from `crypto.getRandomValues(Uint32Array)` with rejection sampling (`limit = 2^32 − (2^32 mod n)`), which is exactly uniform. `Math.random()` remains only as a fallback for environments with no WebCrypto (never a modern browser or Node ≥ 19). No seedable PRNG exists in the codebase, so nothing test-only could leak into the live path. |
| 2 | Settlement (insurance × surrender) | **Medium** | Both surrender branches (`playerAct` and `coachAct` in `src/App.jsx`) set `roundNet = −bet/2` and **dropped `insNet`**. Sequence: dealer shows A → player takes insurance → dealer has no blackjack (insurance loses `bet/2`, held in `insNet` for settlement) → player surrenders (16 vs A *is* a basic surrender). The lost insurance premium was never charged — free insurance. `finalizeOpening` and `resolveRound` both settle `insNet` correctly; only the surrender shortcut missed it. | **Fixed** — both branches now settle `roundNet = −bet/2 + insNet`. |
| 3 | Tests | Low | The `App.dom.test.jsx` smoke test ("plays a Coach Me round") rode on the live unseeded RNG: when the random deal was an instant blackjack (~7 % of deals) the round ends immediately, no Stand/insurance button exists, and the test failed. Observed failing on the first baseline run. | **Fixed** — the assertion now also accepts the round-over ("Change bet") state, making it deterministic across all possible deals. |
| 4 | Shuffle algorithm | — | Fisher–Yates in `buildShoe` iterates `i = n−1 … 1` with `j = rnd(i+1)` — the correct, unbiased form (no `sort(() => random − 0.5)`, no off-by-one). Permutation uniformity verified by chi-square (below). | Verified |
| 5 | Deck integrity | — | `buildShoe` produces exactly 8 × 52 = 416 cards, 32 per rank, 32 aces, 128 ten-values, Hi-Lo tags summing to 0. Monte Carlo asserted per-round conservation (shoe + felt = constant) over 10 M rounds with zero violations. Cut card at 78 cards (~1.5 decks, ≈ 81 % penetration) is realistic for AC; `drawFrom`'s empty-shoe refill is an unreachable emergency guard given the cut card. | Verified |
| 6 | AC rules fidelity | — | See rules table below — all encoded correctly. | Verified |
| 7 | EV tables (`evdata.js`) | Info | Generated by infinite-deck DP, honestly labeled as such (including the two-hand no-resplit split approximation and understated split SD). Spot checks match published values, e.g. 16 vs 10: stand −0.540 / hit −0.540 / surrender −0.500 (so R is correct); soft 18 vs 9: hit −0.101 beats stand −0.183; 8,8 vs 10: split −0.489 beats surrender −0.50 and hit −0.540. Dealer bust table (peek-conditioned S17) matches standard figures incl. the 42 % → 26 % cliff from 6 to 7. Finite-shoe (8-deck) EVs differ from infinite-deck by ≲ 0.005 in these cells — immaterial for a trainer and correctly disclosed. | Verified / disclosed |
| 8 | Count-play numbers | Info | Illustrious-18 indices (16v10 @ 0, 16v9 @ +5, 15v10 @ +4, 10,10 v5/v6 @ +5/+4, 12v2…6, 13v2/3, 10vT/A @ +4, 9v2/v7) and Fab-4 surrender indices (14v10 @ +3, 15v9 @ +2, 15v10 @ 0, 15vA @ +1) match Schlesinger. Insurance threshold TC ≥ +3 is standard. The 11 vs A double deviation (index ≈ +1 for S17) is omitted from the I18 set — a completeness nit, not an error. `edgePct(tc) = 0.5·tc − 0.5` shows −0.5 % off the top vs the true ≈ −0.36 %; a standard rounded teaching heuristic, labelled "Est. edge". | Recommendation |

## AC rules verdict

| Rule | Expected (AC) | Engine | OK |
|------|----------------|--------|----|
| Decks | 8 | `RULES.decks = 8`, `SHOE_CARDS = 416` | ✓ |
| Dealer soft 17 | Stands (S17) | `h17: false`; draw loop `total < 17 || (h17 && soft 17)` | ✓ |
| Blackjack pays | 3:2 | `blackjackPays: 1.5`; unit test asserts +150 on 100 | ✓ |
| Dealer peek | Under 10 and A | `finalizeOpening` peeks when up-card is 10 or A | ✓ |
| Insurance | Offered on ace up, pays 2:1, costs half the bet | `insNet = +bet` on dealer BJ, `−bet/2` otherwise (2:1 on a bet/2 stake); button gated on `balance ≥ 1.5×bet` | ✓ |
| Double | Any two cards | `canDouble = cards.length === 2` (+ bankroll guard) | ✓ |
| Double after split | Allowed | Same `canDouble` applies to split hands (split aces auto-stand, so no DAS on aces — standard) | ✓ |
| Re-splits | To 4 hands | `cg.hands.length < 4` | ✓ |
| Split aces | One card each, no resplit | `isSplitAce` hands auto-complete after one card and can't resplit | ✓ |
| Late surrender | Original two cards only, after peek, returns half | Only when `cards.length === 2 && hands.length === 1 && !isSplitAce`, reachable only after `finalizeOpening` found no dealer BJ; nets −bet/2 | ✓ |
| 21 after split | Pays even money, not 3:2 | Only the original two-card hand triggers the BJ payout path | ✓ |
| All player hands bust | Dealer flips hole card, takes no draws | Explicit branch in `resolveRound`; hole card enters the running count only when revealed | ✓ |

## Verified numbers (Monte Carlo, engine's own code, 10,000,000 rounds)

| Quantity | Measured | Theory | Verdict |
|----------|----------|--------|---------|
| House edge (basic strategy + LS, flat bet) | **0.336 % ± 0.071 %** (95 % CI); a 2 M-round run gave 0.428 % ± 0.158 % | ≈ 0.36 % (8D S17 DAS LS resplit-4; 0.43 % without LS) | ✓ in band |
| Player blackjack frequency | 4.740 % | 4.745 % (= 2·(32/416)·(128/415)) | ✓ |
| Dealer blackjack frequency | 4.739 % | 4.745 % (same combinatorics) | ✓ |
| Dealer bust rate, rounds the dealer played out | 31.1 % | ~28.3 % unconditional S17; conditioning on "no dealer BJ (peek) and ≥ 1 live player hand" removes never-bust BJ rounds and over-samples weak up-cards (player busts more vs 7–A), pushing the conditional rate to ~31 % | ✓ consistent |
| `rnd(52)` chi-square, 2.6 M draws | 45.9 (df = 51) | ≈ 51 | ✓ uniform |
| Fisher–Yates permutations of 4, 240 k shuffles | χ² = 17.4 (df = 23), all 24 orders seen | ≈ 23 | ✓ uniform |
| Shoe composition / conservation | 416 cards, 32/rank, tags sum 0, zero conservation violations in 10 M rounds | exact | ✓ |

**Overall verdict: SOUND after fixes.** The engine's dealt probabilities, payouts, and strategy content
match Atlantic City 8-deck S17 mathematical reality; the realized basic-strategy house edge sits inside the
published 0.36–0.43 % band. The two real defects found (non-crypto RNG source in the live shuffle; dropped
insurance premium on surrender) are fixed in this branch.

## Recommendations (not applied — judgement calls, no correctness defect)

1. **Add the 11 vs A double deviation** (S17 index ≈ +1) if the Illustrious-18 set is meant to be complete.
2. **Label `edgePct`'s −0.5 % baseline** as a rounded teaching value, or use −0.36 % for this exact ruleset.
3. **Consider an injectable RNG for tests** (e.g. a mulberry32 passed only by the test suite) so DOM tests can
   pin specific deals instead of accepting all outcomes; keep it out of the live path per finding 1.
4. **Regenerate `evdata.js` with an 8-deck composition-dependent DP** if you ever want the coach's EV cents to
   be exact for this shoe; current infinite-deck values are within ~0.005 and are honestly disclosed.
